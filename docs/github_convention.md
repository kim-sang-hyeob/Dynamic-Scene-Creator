### 🔀 Github Convention 및 사용절차

### 1. 이슈 생성

1. 이슈를 등록할 때 title 맨 앞에 [이슈 타입] 쓰기  
(예: `[feat] Multi-view 이미지 생성`)

2. 이슈의 description 작성
- 템플릿 사용
```
### 이슈의 SubTask들을 알려주세요.

- [ ] To-do
- [ ] To-do
```

3. 이슈를 담당할 담당자 `Assignees` 지정

4. 이슈 타입에 맞는 `label` 달기

5. 이슈를 등록하면 `#이슈번호`가 할당됨 -> 브랜치명, 커밋 메시지에 사용  
(예: 이슈 제목을 보면 "[feat] Multi-view 이미지 생성 `#10`" <- 이 번호)

<br>

### 2. 브랜치 생성
- 브랜치명 = `타입`/`#이슈번호`
```
feat/#10
```

<br>

### 3. 커밋 시 커밋 메시지 작성
- 커밋 메시지 앞에 `[#이슈번호]` 넣기
  - 이렇게 하면 그 커밋에 관련된 이슈를 확인할 수 있음
```
[#10] 프레임 단위 변경
```

<br>

### 4. Pull requests 날리기
1. PR의 title 맨 앞에 [이슈 타입] 쓰기  
(예: `[feat] SV4D를 통한 Multi-view 이미지 생성 모듈 추가`)

2. PR의 description 작성
- 템플릿 사용
- 이슈 티켓의 resolved 뒤에 #을 치면 해당 이슈 지정가능
```
## Summary
<!-- PR 요약을 써주세요. -->



## Description
<!-- PR에 대한 자세한 설명을 써주세요. -->
- 이슈 티켓 : resolved 


## PR Point
<!-- 코드를 작성할 때 고민했던 부분을 적어주세요 -->


## 📚 Reference 
<!-- 참고할 만한 자료가 있다면 링크나 시각 자료를 달아주세요. -->



## 🔥 Test
<!-- Test -->
```

3. 해당 PR에 대해 리뷰해줄 리뷰어 `Reviewer` 지정

4. 해당 PR의 담당자 `Assignees` 지정

5. 이슈 타입에 맞는 `label` 달기
