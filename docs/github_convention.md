### 🔀 Github Convention 및 사용절차

### 1. 이슈 생성

1. 이슈를 등록할 때 title 맨 앞에 [이슈 타입] 쓰기  
(예: `[feat] Multi-view 이미지 생성`)

2. 이슈의 description 작성
- 템플릿 사용
```
### 이슈의 SubTask들을 알려주세요.

- [ ] To-do
- [ ] To-do
```

3. 이슈를 담당할 담당자 `Assignees` 지정

4. 이슈 타입에 맞는 `label` 달기

5. 이슈를 등록하면 `#이슈번호`가 할당됨 -> 브랜치명, 커밋 메시지에 사용  
(예: 이슈 제목을 보면 "[feat] Multi-view 이미지 생성 `#10`" <- 이 번호)

### 2. 브랜치 생성
- 브랜치명 = `타입`/`#이슈번호`
```
feat/#10
```

### 3. 커밋 시 커밋 메시지 작성
- 커밋 메시지 앞에 `[#이슈번호]` 넣기
  - 이렇게 하면 그 커밋에 관련된 이슈를 확인할 수 있음
```
[#10] 프레임 단위 변경
```
